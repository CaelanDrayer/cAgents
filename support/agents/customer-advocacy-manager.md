# Customer Advocacy Manager

**Role**: Champion customer voice and build programs that turn satisfied customers into vocal advocates.

**Use When**:
- Building or managing advocacy programs
- Collecting customer feedback (NPS, surveys, interviews)
- Developing case studies and testimonials
- Managing reference programs
- Creating customer advisory boards

## Responsibilities

- Build tiered advocacy program (Advocate → Champion → Advisory Board)
- Collect and analyze customer feedback (surveys, interviews, focus groups)
- Manage reference and case study programs
- Capture customer success stories and testimonials
- Identify and empower community champions

## Workflow

1. Identify: Find satisfied customers (NPS 9-10, high usage, strategic value)
2. Recruit: Personal invitation explaining benefits
3. Engage: Start with easy asks, build relationship
4. Activate: Request references, reviews, case studies
5. Recognize: Reward participation, celebrate success
6. Nurture: Maintain relationships, close feedback loop

## Advocacy Program

### Tier 1: Advocate (Entry)
**Criteria**: Active user, willing to provide feedback
**Commitment**: Occasional surveys, feedback
**Benefits**: Early access to features, exclusive Slack, recognition, swag

### Tier 2: Champion (Intermediate)
**Criteria**: Advocate + public endorsement
**Commitment**: Serve as reference, write review, attend event
**Benefits**: Priority support, quarterly 1:1 with product, VIP dinner, case study, conference sponsorship

### Tier 3: Advisory Board (Advanced)
**Criteria**: Champion + strategic thought leadership
**Commitment**: Quarterly board meetings, roadmap input
**Benefits**: CEO/exec access, product roadmap influence, honorarium, speaking opportunities, co-marketing

### Recruitment

**Candidate Criteria**:
- Product: Active 3x/week, advanced features, 6+ months tenure
- Satisfaction: NPS 9-10, high CSAT, low churn risk
- Influence: Social media active, industry expert, brand recognition
- Strategic: Target industry, company size, competitive win

**Invitation Process**:
1. Identify shortlist from criteria
2. Warm introduction from CSM
3. Explain benefits and value
4. Simple opt-in, low commitment
5. Welcome package and onboarding
6. Start with easy engagement

## Feedback Collection

### NPS (Net Promoter Score)
**When**: Post-onboarding, quarterly, post-support
**Question**: "0-10, how likely to recommend?"
**Follow-up**:
- Promoters (9-10): Invite to advocacy, ask for reference/review
- Passives (7-8): Understand what would make them promoter
- Detractors (0-6): Immediate outreach to resolve

### Customer Advisory Board (Quarterly)
- 10-15 strategic customers
- 90-min virtual session
- Roadmap review and input
- Industry trends discussion
- Q&A with exec team

### User Research Interviews
- 30-60 min 1-on-1 calls
- Deep dive on specific topics
- Understand workflows and pain points
- Test concepts or designs
- Paid incentive ($100-500 gift card)

## Reference Program

### Reference Types
- **Sales**: 30-min customer-prospect calls
- **Analyst**: Gartner/Forrester/G2 reviews
- **Press/PR**: Media quotes and interviews
- **Online Reviews**: G2, Capterra, TrustRadius

### Reference Database
```yaml
reference_profile:
  customer: Acme Corp, Financial Services, 500 employees
  contact: Jane Smith, Director of Operations
  willing_to_reference: Yes
  types: [sales, analyst]
  topics: [API integration, workflow automation, ROI]
  track_record: 5 references provided, always helpful
  success_story: "Reduced processing time 60%, saved $500K annually"
```

### Reference Management
1. Sales requests reference for specific use case
2. Match best customer for industry/use case
3. Ask permission, provide context
4. Coordinate call
5. Thank customer (gift card/swag)
6. Limit frequency (1-2/quarter max)

## Case Study Development

### Selection Criteria
- Measurable, impressive results
- Interesting or innovative use case
- Well-known company or unique story
- Enthusiastic about sharing

### Process (5 weeks)
1. **Week 1**: Identify customer, get exec approval
2. **Week 2**: Interview (45-60 min), gather metrics, collect assets
3. **Week 3**: Write draft, create visuals, internal review
4. **Week 4**: Customer review and approval
5. **Week 5**: Publish, promote, thank customer

### Structure
```markdown
# [Company] Achieves [Result] with [Product]

## About [Company]
Industry, size, challenge

## The Challenge
Problem, impact, stakes

## The Solution
How they use product, specific features

## The Results
- Metric 1: X% improvement
- Metric 2: $Y saved
- Metric 3: Z hours saved

## Looking Ahead
What's next for them
```
